# AI潮玩周报制作规范

## 更新时间
2026年4月11日

## 核心要求

### 1. 信息筛选优先级（时间维度）
**严格按时间优先级搜索和筛选：**
- 第一优先级：近一周（7天内）
- 第二优先级：近两周（14天内）
- 第三优先级：近一个月（30天内）

**如果某时间段内产品不足，再向后扩展，不能硬凑10条**

### 2. 产品筛选标准

**必须同时满足以下条件：**

| 条件 | 说明 |
|------|------|
| ✅ 具体产品 | 必须是明确的产品名称，不是行业、趋势、技术 |
| ✅ AI功能明确 | 必须有具体的AI技术或功能，不是单纯的IP联名 |
| ✅ 时间符合 | 发布时间必须在筛选时间范围内 |
| ✅ 信息完整 | 必须有定价/众筹数据/销量等量化信息 |

**排除类型：**
- ❌ 行业新闻/趋势分析（如"AI玩具市场增长"）
- ❌ 消费警示/投诉新闻
- ❌ 展会新闻（除非发布具体产品）
- ❌ 纯技术发布（如芯片、模型开源）
- ❌ 即将发布的产品预告（如"将于X月众筹"）
- ❌ 非AI产品（如纯IP联名盲盒无AI功能）

### 3. 信息格式规范

**每条必须包含：**

```
**序号. 产品名称** [时间标签]
📅 发布时间: （具体日期）
🌐 来源网站: （网站名）
📝 一句话概括: （产品是什么+核心卖点）
🤖 技术栈: （核心技术，用emoji）
💡 创新技术: （具体功能点，不是泛泛而谈）
📈 市场反响: （量化数据：众筹金额/销量/定价/融资额）
```

**时间标签：**
- ⭐本周 = 近7天
- ⭐近两周 = 8-14天
- ⭐近一个月 = 15-30天

### 4. 技术栈emoji规范

| emoji | 含义 |
|-------|------|
| 🤖 | 技术栈（核心技术） |
| 💡 | 创新技术（具体功能） |
| 📈 | 市场反响（数据） |
| 🎨 | 设计/艺术类 |
| 🗣️ | 语音/对话类 |
| ❤️ | 情感/陪伴类 |
| 🧸 | 玩具/硬件类 |
| 🎮 | 游戏/互动类 |

### 5. 创新技术写作要求

**必须具体，避免泛泛而谈：**

❌ 错误示例：
- "采用先进的AI技术"
- "具有情感交互功能"
- "支持多模态识别"

✅ 正确示例：
- "BSLA引擎三重能力：记住吃药时间、主动提醒加衣、构建全场景陪伴"
- "16点仿生皮肤传感器，能区分不同力度和位置的触摸"
- "2秒内根据孩子语调生成个性化童话故事"

### 6. 市场反响量化要求

**必须有具体数字：**

| 类型 | 示例 |
|------|------|
| 众筹 | "已筹$295万，2093名支持者，5小时破百万" |
| 销量 | "累计出货近2万台，70%来自海外" |
| 定价 | "定价6999元，首批限量1000台" |
| 融资 | "完成A轮超千万美元融资" |
| 溢价 | "二级市场溢价近120%" |

### 7. 搜索关键词策略

**按优先级组合：**
1. "AI玩具 新品 众筹 2026年4月"
2. "AI陪伴机器人 新品 2026"
3. "Kickstarter AI toy 2026"
4. "AI潮玩 泡泡玛特 联名"
5. "AI硬件 新品发布 2026"
6. "AI毛绒玩具 可更换人格"

**时间范围设置：**
- 先搜 --time-range week（近一周）
- 不足再搜 --time-range month（近一个月）

### 8. 本周经验教训

**问题：**
1. 最初为了凑够10条，混入了不符合标准的信息
2. 时间范围把控不严，把3月的产品当成本周新品
3. 对"具体产品"的判断标准不够严格

**改进：**
1. 严格按照"近一周→近两周→近一个月"优先级筛选
2. 宁可少而精，不要凑数
3. 每条信息必须验证：是否具体产品？是否有AI功能？时间是否符合？
