# Minecraft 服务器繁荣度系统项目记录

## 项目愿景
打造一个"由玩家文明驱动演化的虚拟世界"，在《我的世界》中构建MMORPG式的永续社会。

## 核心架构：三层服务器模型
- **主服务器**: 永恒的家园与文明主体
  - 模组绝对固定（初期锁定核心模组）
  - 核心玩法：副本 + 文明演化系统
  - 副本定义：主世界内或独立地图的剧情与进度挑战
  
- **周目服**: 独立运行的赛季玩法试验区  
  - 独立存在，定期更换主题模组包（魔法季、科幻季等）
  - 测试玩法、引流、提供额外游戏目标
  - 产出以荣誉性、装饰性方式反哺主服
  
- **休闲区**: 主服内的社交枢纽
  - 内置于主服务器安全区的常设小游戏集合

## 主服务器两大支柱系统

### 1. 阶段性剧情推进系统（通过"副本"实现）
- 流程：玩家挑战副本来达成阶段目标，解锁全服新内容
- 关键机制：
  - 装备复制挑战：凭主世界装备资格进入，副本内使用复制品，无损挑战
  - 进度共享：阶段解锁后，全服玩家自动获得上一阶段的便利性福利

### 2. 文明演化系统（通过"组织"与"繁荣度"实现）
- 组织多元：正式认可"城镇"、"公会"等多种玩家集体形态
- 繁荣度算法：由社区共识驱动，包括：
  - 公共建筑的玩家点赞量
  - 自创历史的认可度  
  - 成员规模与副本进度等

## 技术与管理策略
- 核心原则："主服模组固定如山，周目服模组灵活轮换"
- 运营角色：管理团队作为"世界架构师"与"史官"
